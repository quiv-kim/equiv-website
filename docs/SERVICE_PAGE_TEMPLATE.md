# EQUIV Service Page Template

이 문서는 EQUIV 일반 서비스 상세페이지의 공식 템플릿이다.

기업 매각 페이지를 Master Template로 사용한다. 서비스마다 업무 내용은 달라도 사용자는 어느 페이지에서나 같은 정보 흐름과 브랜드 경험을 만나야 한다. `business-valuation-review.html`의 기업가치 검토 상세페이지는 이 템플릿을 따르며, 사전진단 전용 `business-valuation.html`은 별도 기준을 따른다.

## Core Principle

서비스를 길게 설명하지 않는다. 대표가 자신의 상황에 필요한 도움인지 판단할 수 있도록 거래 과정과 문제 해결을 먼저 보여준다.

- 한글을 중심에 두고 영문은 서비스 구분을 보조한다.
- 실제 업무를 짧고 구체적인 문장으로 작성한다.
- 과장, 결과 보장, 가격 경쟁과 추상적인 홍보 표현을 사용하지 않는다.
- 기존 Typography, Grid, Spacing, Color, Animation과 Component만 사용한다.

## Official Page Structure

Header와 Footer를 제외한 모든 일반 Service Page는 아래 순서를 유지한다.

1. Hero
2. Overview
3. How We Work
4. FAQ
5. CTA

섹션을 임의로 추가하거나 순서를 바꾸지 않는다. 새로운 정보가 필요하면 기존 다섯 섹션 안에서 짧게 정리한다.

## 1. Hero

서비스의 성격과 대표 관점을 전달하는 첫 영역이다.

- Gold 영문 서비스명, 한글 H1, 한글 Headline 순서를 사용한다.
- Headline은 사용자의 상황과 판단 기준을 한 문장으로 표현한다.
- 별도의 장문 설명은 두지 않는다.
- HOME Hero보다 작고 단정한 기존 배경과 타이포그래피를 유지한다.

## 2. Overview

서비스가 왜 필요한지 설명하고 EQUIV의 역할을 짧게 정의한다.

- 라벨은 `Overview`를 사용한다.
- 한글 Headline과 본문 1~2개 문단으로 구성한다.
- 서비스 범위를 나열하지 않고 사용자의 거래 목적과 의사결정에서 시작한다.
- 같은 의미를 반복하지 않는다.

## 3. How We Work

EQUIV가 실제로 거래를 진행하는 방식을 보여주는 핵심 섹션이다.

- 라벨은 `How We Work`를 사용한다.
- 네 단계의 기존 Line Process를 사용한다.
- 각 단계는 한글 업무명과 한 문장 설명으로 구성한다.
- 설명은 `검토합니다 / 정리합니다 / 발굴합니다 / 연결합니다 / 조율합니다 / 지원합니다`처럼 실제 업무 동사를 사용한다.
- 새로운 카드나 장식, 애니메이션을 추가하지 않는다.

기업 매각 기준 단계:

1. 기업 이해
2. 거래 전략
3. 매수자 발굴
4. 협상 및 거래 종결

## 4. FAQ

대표와 의사결정자가 실제 상담 전에 자주 묻는 질문을 다룬다.

- 라벨은 `FAQ`를 사용한다.
- 서비스별 핵심 질문 네 개를 기준으로 한다.
- 비슷한 질문은 합치거나 삭제한다.
- 답변은 단정하거나 결과를 보장하지 않고 진행 기준과 방식을 설명한다.

## 5. CTA

페이지에서 형성된 신뢰를 상담 검토로 연결한다.

- 기존 Navy CTA 디자인을 유지한다.
- 서비스별 질문형 Headline, 한 문장의 Body와 구체적인 상담 버튼을 사용한다.
- 긴급성, 보장, 무료와 과장 표현을 사용하지 않는다.

기업 매각 기준:

- Headline: `기업 매각을 검토하고 계십니까?`
- Body: `거래 가능성과 전략을 함께 검토해 보십시오.`
- Button: `기업 매각 상담`

## Shared Design Rules

- 기존 Container, Grid, Section Spacing과 반응형 기준을 유지한다.
- Navy / White 중심으로 사용하고 Gold는 라벨과 포인트에만 사용한다.
- 새로운 카드, 아이콘, 강한 그림자와 애니메이션을 추가하지 않는다.
- Desktop, Tablet, Mobile에서 같은 섹션 순서를 유지한다.
- Header, Footer와 Back to Top 구조를 공통으로 유지한다.

## Content Review Checklist

- [ ] 본문이 공식 5개 섹션 순서를 따르는가
- [ ] 서비스 설명보다 사용자의 고민과 거래 과정이 먼저 보이는가
- [ ] Hero에 장문 설명이 없는가
- [ ] Overview가 짧고 중복 없이 작성되었는가
- [ ] How We Work가 네 단계의 실제 업무를 보여주는가
- [ ] FAQ가 핵심 질문 네 개로 정리되었는가
- [ ] CTA가 짧고 서비스별 행동을 분명히 안내하는가
- [ ] 과장, 보장과 가격 중심 표현이 없는가
- [ ] Desktop, Tablet, Mobile에서 가독성과 정보 흐름이 유지되는가
