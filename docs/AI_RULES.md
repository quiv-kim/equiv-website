# EQUIV AI Development Rules

## Purpose

이 문서는 EQUIV 프로젝트에서
ChatGPT, Codex 및 향후 참여하는 모든 AI가
반드시 따라야 하는 프로젝트 개발 원칙이다.

이 문서의 목적은
브랜드의 일관성과 프로젝트 방향성을 유지하는 것이다.

# Rule 1

브랜드 철학을 우선한다.

코드는 브랜드를 표현하기 위한 수단이다.

브랜드 철학보다 디자인을 우선하지 않는다.

# Rule 2

EQUIV는 M&amp;A Advisory Firm이다.

기업가치평가는 강점이지만
홈페이지가 가치평가 회사처럼 보여서는 안 된다.

항상

M&amp;A

↓

거래 전략

↓

기업가치

순서로 표현한다.

# Rule 3

홈페이지는 영업자료다.

단순히 예쁜 홈페이지를 만들지 않는다.

모든 페이지는

대표가

"이 회사와 상담해보고 싶다."

라는 생각이 들도록 만든다.

- 최고의

# Rule 5

Premium를 유지한다.

Gold를 과도하게 사용하지 않는다.

여백을 적극적으로 사용한다.

애니메이션은 최소화한다.

# Rule 6

한글 중심

영문 보조

핵심 메시지는 항상 한글이 먼저다.

# Rule 7

One Page가 아니다.

Corporate Website로 개발한다.

HOME은 요약이다.

상세 설명은 Detail Page에서 한다.

# Rule 8

새 기능보다 완성도를 우선한다.

기능을 계속 추가하지 않는다.

현재 있는 페이지의 품질을 먼저 높인다.

## Pricing Rule

새로운 페이지나 콘텐츠를 작성할 때
다음 내용을 임의로 작성하지 않는다.

- 착수금 없음
- 성공보수 %
- 수수료 %
- 무료
- 최저 비용
- 할인
- 리베이트
- 파트너 수수료
- 비용 비교 문구

가격 관련 내용은
사용자가 명시적으로 요청하거나
별도의 계약 안내 페이지를 제작하는 경우를 제외하고
홈페이지에서 먼저 제안하지 않는다.

서비스는
가격보다
전문성,
거래 전략,
브랜드 철학,
업무 방식,
신뢰를 중심으로 설명한다.

## Preliminary Valuation Rule

웹사이트와 공개 콘텐츠에서는 다음 내용을 표시하지 않는다.

- 확정 기업가치
- 확정 매각가격
- 보장된 거래가격
- 정식 기업가치평가로 오인될 수 있는 자동 산출 결과

다만 아래 조건을 모두 충족하는 경우 예비 기업가치 범위 산출 기능을 공개할 수 있다.

- 사용자가 입력한 제한적인 기초정보를 기반으로 한다.
- 단일 금액이 아닌 하단·중심·상단의 범위로 표시한다.
- 정식 기업가치평가, 세무상 평가액 또는 실제 매각가격이 아님을 명확히 표시한다.
- 업종별 배수, 가중치와 보정계수는 별도 설정 파일에서 관리한다.
- 계산할 수 없거나 신뢰도가 낮은 경우 임의 금액 대신 추가 검토가 필요하다고 표시한다.
- 공개 진입점은 Business Valuation 페이지로 제한하고 HOME에는 예비진단 버튼을 추가하지 않는다.
- 사용자 화면에는 내부 테스트, 개발 버전과 Calibration 설정값을 노출하지 않는다.

공개용 예비진단도 입력정보와 결과를 서버 또는 외부 데이터베이스로 전송하거나 저장하지 않는다.

# Final Principle

## Official Logo Rule

EQUIV의 공식 Primary Logo는 `The EQUIV Mark + Vertical Divider + EQUIV`의 고정 Lockup이다.

- Header, Footer와 Consultation Modal은 동일한 Lockup을 사용한다.
- 기본 로고에 `M&A ADVISORY`, `M&A Brokerage` 등 보조 설명을 결합하지 않는다.
- Favicon은 The EQUIV Mark 심볼만 사용한다.
- Mark, Divider와 Wordmark의 비율을 임의로 변형하거나 별도의 텍스트 로고로 대체하지 않는다.
- Gold는 Mark와 제한적인 강조에만 사용하며 과도한 Gradient를 추가하지 않는다.

# Final Principle

의심이 생기면

더 화려한 디자인보다

더 단순한 디자인을 선택한다.

더 많은 문장보다

더 좋은 문장을 선택한다.

더 많은 기능보다

더 높은 완성도를 선택한다.
